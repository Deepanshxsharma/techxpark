# Contributing to TechXPark

First off, thank you for considering contributing to TechXPark! 

### 1. Where do I go from here?
If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME/issues) to see if someone else has already created a ticket. If not, go ahead and make one!

### 2. Fork & create a branch
If this is something you think you can fix, then fork TechXPark and create a branch with a descriptive name.

### 3. Implement your fix or feature
Ensure your code follows the existing style, uses the defined `AppColors` and `AppTextStyles`, and runs without errors.
Before bringing your changes into the main codebase, run:
```bash
flutter analyze
```

### 4. Make a Pull Request
At this point, you should switch back to your master branch and make sure it's up to date with TechXPark's master branch.
Then push your feature branch to your fork, and submit a pull request!
